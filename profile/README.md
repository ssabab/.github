## 🎯 2. Golden Circle 정리 (기존)

![Project Goal.png](attachment:96d4e663-77e1-4cf4-81c0-5fa3668e3e63:Project_Goal.png)

### Why (목적)

> 싸피 교육생들이 반복적인 점심 메뉴 선택 스트레스 없이,
> 
> 
> 자신의 식사 취향을 발견하고 더 나은 선택 경험을 할 수 있도록 돕는다.
> 

### How (what을 제공하기 위한 방법)

1. 오늘의 점심 메뉴(A/B) 정보를 명확히 제공하고, 선택을 돕는다.
2. 개인의 식사 기록을 저장하고, 선택 패턴을 기반으로 다음 메뉴를 추천한다.
3. 친구와의 선택 비교를 통해 식사 경험에 소셜 요소를 더한다.

### What (기능)

- 메뉴 정보 제공 (구성, 국물/맵기 등 태그 표시)
- A/B 메뉴 선택 및 기록 기능
- 식사 후 별점 및 한줄평 작성 기능
- 개인화 추천 (선택 패턴 분석)
- 친구 추가 및 선택 패턴 비교 기능

### 6-2. 기능 확장 전략

### 6-3. 홈화면이 필요한 이유

> “기능이 늘어날수록, 사용자에게는 **흐름을 안내해줄 중심 공간**이 필요하다.”
> 

---

1. **사용자에게 하루 식사 흐름을 ‘타임라인’처럼 연결해주는 허브**
    - 점심 메뉴 → 카페 추천 → 저녁 외식 추천처럼, 사용자 하루 식사 활동이 늘어나는 구조에선
        
        **탭마다 따로따로 정보가 있으면 서비스가 산만하게 느껴짐**
        
    - 홈화면이 있어야
        
        **"오늘 내가 어떤 선택을 해야 하지?"**, **"지금 어떤 정보가 준비되어 있지?"** 를 한눈에 확인할 수 있음
        
        ---
        
2.  **기능이 확장될수록 중심이 되는 ‘메인 경험’이 필요함**
    - 초기에는 점심 메뉴만 선택했지만,
        
        카페 추천, 저녁 추천, 친구 선택 피드백, 개인 리포트 등 **기능이 늘어나면** 
        
        홈은 그 기능들을 **필요한 타이밍에, 필요한 순서로 자연스럽게 제시해주는 안내판 역할**을 함
        
    
    ---
    
3. **정보의 우선순위를 ‘콘텐츠 흐름’으로 보여줄 수 있음**
    - 홈에서는 탭보다 더 유연하게
        
        **“중요한 정보 먼저, 덜 중요한 정보 나중에”** 보여줄 수 있음
        
        (예: 오늘 메뉴 → 추천 → 나의 기록 → 친구 피드백 등)
        
    - 탭 방식은 구조적이지만,
        
        홈화면은 콘텐츠 기반의 흐름이 가능해서 **선형적 경험을 설계하기 좋음**
        

### 6-3. 친구 탭 마이페이지로 병합

- 친구 기능은 **UI에서 명확한 진입점이 반드시 있어야 함**
- 그게 **친구 탭**이 아니더라도
    
    👉 예: 마이페이지 안 ‘친구 관리’
    
    👉 예: 또는 홈 하단 ‘친구의 오늘 선택’ 안 ‘+ 친구 추가’ 버튼 **등의 대체 경로가 필요함.**
    
- 인스타그램 계정을 보는 것 같은, 그러한 느낌의 UI로 진행하면 괜찮을 것 같음
- 듀오링고도 마이페이지에 친구 목록이 함께 존재

## 7. ❓의문점 (사이트를 못써봐서 의문이 드는 것일수도 . . .)

- **식단 선택과 실제 섭취가 다를 경우 데이터 처리 방식**
    - A 식단을 선택했는데 수량이 없어 B 식단을 실제로 먹은 경우
        - B에 투표할 시 → B를 선호해서 먹은 것으로 인식될 수 있음
    - 이 경우 어떤 기준으로 기록할지?
    - 예: 식사 전 선택 데이터와 식사 후 실제 섭취 데이터 분리해서 기록할지 여부도 궁금함

---

- **식사 전·후 선택 분리 여부**
    - 식사 전: 기대 메뉴 선택
    - 식사 후: 실제 먹은 메뉴에 대해 별점, 한줄평 등 피드백
    - 위 두 선택을 분리해서 다루는지, 아니면 하나의 흐름으로 통합할지 결정 필요함

---

- **홈화면 ‘최근 나의 선택 기록’ 표시 방식 → 주메뉴를 보여주는 방식 채택**
    - 기록 요약을 태그 기준으로 보여줄지, 주요 메뉴 이름 기준으로 보여줄지 고민됨
        - 태그 기준: 사용자가 정확히 뭘 먹었는지 기억하기는 어려우나 속성 파악은 쉬움
        - 주요 메뉴 기준: 한눈에 기억은 쉬우나, 관리자가 주요 메뉴를 따로 입력해야 하는 부담 있음
    - 어떤 방식이 더 적절한지, 혹은 혼합 방식도 고려 가능한지 확인 필요함
